1. 추진 배경 및 핵심 방향

본 과제의 목적은 기존 계정 거래 페이지의 단순 상품 나열 구조를 구매 판단을 돕는 인사이트형 구조로 전환하여, 거래 전환율과 시장 신뢰도를 동시에 높이는 것입니다.

기존 플랫폼이 ‘상품 나열’에 머물렀다면, 본 구조는 ‘상품 + 해석 + 판단’을 제공하여 구매자는 판단 가능성을 얻고, 판매자는 가격 교정 기준을 얻으며, 플랫폼은 신뢰를 확보하는 방향으로 설계해야 합니다. 또한 프로토타입을 기반으로 실제 서비스 수준까지 정교화하고, 게임마스터 점검을 포함한 검증 구조를 반드시 갖춰야 합니다.
핵심 인식 전환

거래 플랫폼 → 판단 플랫폼

가격과 스펙을 함께 해석하여 사용자가 지금 사도 되는지, 비싼지 싼지, 왜 그런지까지 이해할 수 있는 구조가 필요합니다.

사용자 관점

정보 비대칭 해소

복잡한 계정 거래일수록 적정가와 구성 해석 수요가 크며, 이 지점을 해결할수록 체류와 구매 전환이 함께 올라갈 가능성이 높습니다.

운영 관점

리포트는 전환 장치

AI 자체를 전면에 내세우기보다 가격·구성·근거·리스크를 짧고 확신형 문장으로 제시해 클릭 이후 구매 행동을 유도해야 합니다.

회장님 공유 방향: 현재 화면은 완성본이 아닌 프로토타입이며, 이후 단계에서는 카드 구조, 가격 판단 로직, 문구 기준, 검증 체계를 포함한 정교화가 필요합니다.

2. 대상 게임 선정 방향

대상 게임은 거래량 자체보다 ‘AI 리포트가 실제로 먹히는 구조인지’를 기준으로 선정해야 하며, 가격 편차·스펙 복잡도·정보 비대칭·시장 데이터 확보 가능성을 함께 봐야 합니다.

게임 평가 선정 이유 리스크 실행 판단
리니지M 1순위 필수 가격 편차가 크고, 변신/인형/문양/악세 등 스펙 해석 요소가 많아 AI 리포트 가치가 가장 높음 리포트 로직 정교화 필요 1차 MVP 즉시 적용
리니지클래식 2순위 추천 시장 초기로 정보 비대칭이 크고, 기준 가격 제시 역할이 가능하여 리포트 구조가 효과적으로 작동할 가능성이 높음 완료 데이터 부족으로 적정가 정확도 보완 필요 리니지M과 함께 1차 적용
아이온2 조건부 보류 거래 활성화 흐름은 있으나 가격 구조와 스펙 해석 복잡도가 상대적으로 낮아 ‘AI 적정가 리포트’ 수요가 약할 가능성 존재 분석보다 추천 매물 큐레이션이 더 잘 먹힐 수 있음 2차 테스트 대상, 추천형 카드 중심 검토
보고 기준상 1차 대상 게임은 리니지M, 리니지클래식으로 정리하고, 아이온2는 추천형 테스트로 별도 검토하는 방향이 가장 현실적입니다.

3. 서비스 적용 구조

메인에서는 빠른 판단을 유도하는 카드형 섹션으로, 상세에서는 적정가와 구성 근거를 설명하는 리포트형 화면으로 역할을 분리합니다.

메인 섹션

매니아 인사이트 상품

  • 배너 아래, 상품리스트 위에 배치
  • 카드 3개 구성: 저평가 / 적정가 / 고가
  • 핵심 노출: 게임명, 서버/클래스, 가격, 한줄 분석, 스펙 3개
  • CTA 구성: 리포트 보기 / 구매하기
  • 목적: 3초 내 주목 매물 판단 유도
상세 리포트

전문가 적정가 분석

  • 현재 판매가와 적정가 범위 시각화
  • 가격 근거: 핵심 스펙, 희소성, 패키지 가치
  • 리스크 요소: 성장 여지, 서버 변수, 구성 한계
  • 추천 대상: 즉시 플레이형 / 과금 최소화형 등
  • 유사 매물 비교 및 거래 예상 기간 제시
대외 노출 문구는 ‘AI’보다 ‘전문가/자문단/심사단 분석’ 톤으로 운영하고, 내부적으로는 AI 기반 산정 로직을 활용하는 이원화 정책이 필요합니다.
현재 카드형 구조는 프로토타입 기준으로 방향성이 확인된 상태이며, 이후 실제 적용 시에는 가격 가독성, 문구 신뢰도, 배지 규칙, CTA 우선순위, 모바일 반응형까지 함께 정교화해야 합니다.

4. 운영 기준 및 데이터 정의

초기에는 완전한 AI 모델보다 룰 기반 산정 + 운영 검수 구조로 시작하고, 성과 데이터가 쌓이면 자동화 범위를 넓히는 것이 현실적입니다.

카드 노출 기준
  • 저평가: 동일 스펙 대비 가격 매력도 상위
  • 적정가: 거래 가능성이 높고 평균 범위 내
  • 고가: 시세 상단이나 희소성/완성도 높은 매물
카드 필수 데이터
  • game / server / class / level
  • price / price_label
  • summary / top_features 3개
  • thumbnail / item_id / CTA 링크
리포트 필수 데이터
  • ai_price_min / ai_price_max
  • risk_points / recommended_for
  • similar_items / expected_sell_days
  • 거래 완료 기준 시세 구간
가격 산정 원칙
  • 기준 시세: 최근 거래 완료 매물 기반
  • 가산 요소: 변신/인형, 문양, 악세, 희귀 구성, 부캐 포함 여부
  • 감산 요소: 상세 정보 부족, 시장 수요 약세, 성장 여지 제한
  • 최종 산출: 적정가 범위 + 저평가/적정가/고가 라벨
경쟁사 활용 기준
  • 바로템 등 경쟁사 가격은 우선 내부 비교 로직에 반영
  • 초기 사용자 노출은 ‘시장 비교 기준’ 수준으로 순화
  • 직접 비교 노출은 법무/운영 검토 후 2차 판단

4-1. 신뢰 기반 검증 구조

회장님 공유 사항에 따라 현재 화면은 프로토타입으로 보고, 실제 서비스 전개 시에는 AI 분석에 더해 게임마스터 점검까지 포함한 검증 구조를 갖춰야 합니다.

프로토타입 고도화
  • 현재 카드형 시안은 방향 검증용 프로토타입으로 활용
  • 실서비스 반영 전 가격/문구/스펙 표기 기준 정교화
  • 게임별 표현 차이와 예외 케이스 정리 필요
  • 추천 근거 문구와 상세 리포트 연결 구조 고도화
게임마스터 점검
  • AI 및 운영자 1차 판단 후 게임마스터 검수 진행
  • 직업/서버/옵션/세팅 가치 판단의 정확도 보완
  • 유저 체감과 다른 가격/문구 노출 리스크 최소화
  • 검수 통과 매물 중심으로 메인 노출 및 리포트 운영
즉, 본 과제는 ‘AI 기능 추가’가 아니라 ‘프로토타입을 정교화하고 게임마스터 점검까지 거친 검증형 거래 판단 구조’로 발전시키는 프로젝트로 정의해야 합니다.

5. KPI 및 성과 측정 기준

본 과제의 성과는 단순 노출이나 CTR만으로 평가하면 안 되며, 리포트 진입 후 실제 구매 행동 연결 여부까지 봐야 합니다.

노출 단계
  • 카드 CTR
  • 전체보기 클릭률
  • 게임별 카드 반응 차이
판단 단계
  • 리포트 상세 진입률
  • 리포트 체류시간
  • 리포트 내 버튼 클릭률
전환 단계
  • 리포트 진입 후 구매 클릭률
  • 리포트 노출 매물 판매 전환율
  • 일반 매물 대비 판매 소요기간 단축률
성과 정의는 ‘카드 클릭’이 아니라 판단 보조 구조가 거래 연결에 실제 기여했는지를 기준으로 관리해야 하며, 검증 반영 매물과 일반 매물 간 성과 차이도 함께 확인해야 합니다.

6. 단계별 실행 로드맵

초기에는 리니지M·리니지클래식 중심의 MVP를 빠르게 띄우고, 데이터와 반응을 확인한 뒤 타 게임과 추천형 구조로 확장하는 순서가 적합합니다.

STEP 11주차

기획 확정 및 운영 룰 정의

  • 섹션명, 문구 정책, 카드 필드, 리포트 필드 최종 확정
  • 저평가/적정가/고가 판정 기준 정의
  • 리니지M, 리니지클래식 대상 게임 확정
  • 외부 노출 문구를 전문가/심사단 톤으로 확정
STEP 22주차

디자인 시안 및 프로토타입 고도화

  • 회장님 카드형 프로토타입 기준 메인 섹션 디자인 확정
  • 프로토타입을 실제 서비스 수준으로 정교화
  • 상세 리포트 내 적정가 범위 시각화 UI 설계
  • 모바일 카드 축약 버전 및 상세페이지 반응형 구조 설계
  • 리니지M/리니지클래식 각각 예시 매물 시안 제작
STEP 33주차

MVP 개발 및 데이터 연결

  • 메인 카드 API 및 상세 리포트 API 개발
  • 카드 선정 로직과 리포트 데이터 매핑 적용
  • 배너 아래 / 상품리스트 위 영역에 메인 섹션 삽입
  • 리포트 페이지 또는 오버레이 연결
STEP 44주차

운영 테스트 및 성과 검증

  • 리니지M, 리니지클래식 대상 20~30개 매물 수동·반자동 운영
  • 카드 CTR, 리포트 진입률, 구매 클릭률 집중 점검
  • 저평가/적정가/고가 문구 반응 비교
  • 게임마스터 점검 프로세스 병행 운영
  • 시장 반응이 약한 문구 및 위치 즉시 수정
STEP 55~6주차

확장 여부 판단 및 2차 모델 기획

  • 성과 양호 시 타 게임 확장 검토
  • 아이온2는 AI 적정가형보다 추천 매물형 별도 테스트 검토
  • 경쟁사 비교 반영 범위와 자동화 수준 재정의
  • 성과 미흡 시 카드 구조·카피·운영 기준 재조정

7. 최종 보고 포인트

1차 실행은 리니지M과 리니지클래식을 대상으로 메인 카드형 ‘매니아 인사이트 상품’ 섹션과 상세 ‘전문가 적정가 분석’ 리포트를 구축하고, 프로토타입을 정교화한 뒤 게임마스터 점검을 반영한 검증형 운영 구조를 갖추고, 4주 내 MVP를 통해 거래 연결 효과를 검증하는 방향으로 추진하는 것이 가장 현실적입니다.

보고 시 강조 포인트

  • 이 과제는 단순 UI 개선이 아니라 거래 판단 구조 전환 프로젝트임
  • 게임 선정이 성패를 좌우하므로 1차 대상은 리니지 계열로 한정 필요
  • AI는 내부 산정 로직으로 활용하고, 외부 노출은 전문가 분석으로 운영
  • 프로토타입 단계 이후 정교화 및 게임마스터 점검이 핵심 성공 조건임

의사결정 요청 사항

  • 1차 대상 게임: 리니지M / 리니지클래식 확정 여부
  • 메인 섹션 명칭 및 문구 톤 확정 여부
  • 게임마스터 점검 프로세스 운영 여부
  • 4주 MVP 추진 일정 승인 여부
