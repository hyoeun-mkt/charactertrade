사용 방법

아래 프롬프트를 복사해서 AI에 붙여넣은 뒤, 스크린샷과 텍스트를 함께 제공합니다.

STEP 01
프롬프트 복사
아래 전체 프롬프트를 복사합니다.
STEP 02
AI에 붙여넣기
Claude / GPT-4o / Gemini 중 어디든 붙여넣습니다.
STEP 03
파일 첨부
같은 상품의 스크린샷(png)과 텍스트를 함께 첨부합니다.
STEP 04
결과 저장
출력된 테이블을 엑셀/스프레드시트에 붙여넣어 누적합니다.
⚠️
파일명 규칙: 같은 상품의 스크린샷과 텍스트는 확장자만 다르고 파일명이 동일해야 합니다. 예: 260309_brt_0001.png + 260309_brt_0001.txt

바로템 페이지 섹션 구조

전체 스크린샷은 아래 순서로 섹션이 세로 나열됩니다. 섹션 순서를 기준으로 읽으세요.

순서섹션명추출 항목위치 특징
1상품 기본 정보클래스·서버·거래완료가·레벨최상단. 텍스트 정보 중심. 레벨은 캐릭터 초상화 왼쪽 하단 원형 뱃지 안 숫자.
2장비 정보이동악세 수량·각인 여부·문양 완성도캐릭터 전신 + 장비 슬롯 패널. 가로로 아이콘 나열.
3~N변신 패널 (반복)변신 등급·각성 여부·옵션원형 아이콘 1개가 중앙에 크게. 주변에 작은 원들이 방사형 배치.
N+1인형 패널 (반복)인형 등급·각성 여부·옵션변신 패널과 동일 구조. 아이콘 형태 다름.
N+2성물 패널 (반복)성물 등급·각성 여부변신 패널과 동일 구조.
N+3스킬 목록신화스킬 수·전설스킬 수표 형태. 스킬명과 등급 아이콘 나열.
N+4컬렉션·문양컬렉션 진행도·문양 완성도최하단부.

📋 전체 프롬프트 (복사해서 사용)

아래 전체 내용을 복사해서 AI에 붙여넣은 뒤 스크린샷과 텍스트 파일을 첨부합니다.

[역할] 너는 리니지M 계정 거래 데이터 분석 전문가다. 바로템 거래완료 페이지의 스크린샷(이미지)과 텍스트를 함께 분석해서 아이템매니아 상품 추천 시스템의 시세 DB와 리포트 생성을 위한 구조화된 데이터를 추출한다. [목적] 1. 시세 기준값 구축: 거래완료 데이터를 누적해 클래스별 최저·평균·최고가 산출 2. 리포트 생성: 상품 상세 리포트(가격 위치·스펙 평가·활용 가능성)에 필요한 문장 생성 [입력 데이터] - 텍스트 파일: 거래완료가·서버·클래스·판매자 설명 포함 - 스크린샷 이미지: 바로템 거래완료 페이지 전체 캡처 (파일명이 같고 확장자만 다른 것이 같은 상품임) [이미지 분석 원칙 — 반드시 준수] 이미지는 바로템 거래완료 페이지를 전체 스크롤 캡처한 것이다. 전체 이미지를 한 번에 읽으려 하지 말고, 아래 섹션 순서대로 섹션 하나씩 집중해서 세밀하게 읽어야 한다. 이미지 해상도가 낮을 수 있으므로 각 섹션의 아이콘 위치·색상·개수를 최대한 정밀하게 파악한다. 확인이 불가한 항목은 반드시 "불명"으로 표기하고 절대 임의 추정하지 않는다. [레벨 확인 방법 — 중요] 레벨은 아래 위치에 표시된다. 반드시 이 부분을 집중해서 읽어라. 위치: 이미지 상단부 섹션 1 내, 캐릭터 초상화(얼굴 이미지) 왼쪽 하단 형태: 원형 뱃지(동그라미) 안에 숫자로 표시 예시: 원 안에 "89"라고 적혀 있으면 레벨 89 주의: 캐릭터 초상화 위쪽이나 오른쪽이 아닌 반드시 왼쪽 하단 원형 안 숫자 [바로템 페이지 섹션 구조 — 섹션별 순서대로 읽기] 이미지는 세로로 스크롤되는 전체 페이지 캡처다. 섹션이 가로선으로 구분되어 아래 순서로 나열된다. 반드시 섹션 1부터 순서대로 하나씩 집중해서 읽는다. 섹션 1 — 상품 기본 정보 (최상단) 추출 항목: 클래스 / 서버 / 거래완료가 레벨: 캐릭터 초상화 왼쪽 하단 원형 뱃지 안 숫자 (위 [레벨 확인 방법] 참고) 읽기 방법: 텍스트가 중심이므로 텍스트 파일과 함께 교차 확인 섹션 2 — 장비 정보 추출 항목: 이동악세 수량 / 각인 여부 / 문양 완성도 특징: 캐릭터 전신 + 장비 슬롯 아이콘이 가로로 나열 읽기 방법: 악세서리 슬롯 위치의 아이콘 수량을 세어 이동악세 수량 파악 섹션 3~N — 변신 패널 (섹션마다 1개씩 반복) 추출 항목: 변신 등급 / 각성 완료 여부 / 각성 옵션 특징: 원형 아이콘 1개가 중앙에 크게, 주변 작은 원 방사형 배치 읽기 방법: 아래 [색상 → 등급 변환 규칙] 적용하여 등급 판단 섹션 N+1 — 인형 패널 (반복) 추출 항목: 인형 등급 / 각성 완료 여부 / 옵션 읽기 방법: 변신 패널과 동일하게 색상으로 등급 판단 섹션 N+2 — 성물 패널 (반복) 추출 항목: 성물 등급 / 각성 완료 여부 읽기 방법: 변신 패널과 동일하게 색상으로 등급 판단 섹션 N+3 — 스킬 목록 추출 항목: 신화스킬 수 / 전설스킬 수 특징: 표 형태로 스킬명·등급 아이콘 나열 읽기 방법: 아이콘 색상으로 신화스킬·전설스킬 구분 후 각각 개수 집계 섹션 N+4 — 컬렉션·문양 (최하단) 추출 항목: 컬렉션 진행도 / 문양 완성도 [색상 → 등급 변환 규칙 — 핵심] 각 섹션에서 아이콘의 색상 종류와 개수를 파악하여 아래 규칙으로 텍스트 변환한다. ※ 리니지M 게임 화면 자체의 색상 기준 (바로템 UI 색상 아님) 색상 | 리니지M 등급 | 변환 예시 -------------|-------------|------------------------------ 흰색/회색 | 일반 | 흰색 3개 → "일반 3개" 초록색 | 고급 | 초록색 2개 → "고급 2개" 파란색 | 희귀 | 파란색 1개 → "희귀 1개" 보라색 | 영웅 | 보라색 2개 → "영웅 2개" 금색/노란색 | 전설 | 금색 3개 → "전설 3개" 붉은색/주황 | 신화 | 붉은색 1개 → "신화 1개" 무지개/특수 | 유일 | 무지개 1개 → "유일 1개" 적용 방법: 1. 각 섹션(변신·인형·성물·스킬)에서 보이는 아이콘 색상을 모두 확인 2. 같은 색상끼리 개수를 센다 3. 위 표에 따라 "등급 N개" 형태로 텍스트 변환 4. 여러 등급이 섞여 있으면 모두 나열 예: 금색 2개 + 보라색 1개 → "전설 2개, 영웅 1개" 각성 완료 판단: 아이콘 주변 날개형 이펙트 또는 강한 빛 테두리 있음 → 각성 완료 이펙트 없음 → 각성 미완 판단 불가 → 불명 [리니지M 게임 내 등급 기준] ※ 바로템 UI 색상이 아닌, 이미지 안 리니지M 게임 화면 자체의 색상 기준 등급 | 아이콘 색상·특징 --------|------------------------------------------ 일반 | 흰색/회색 테두리 고급 | 초록빛 테두리 희귀 | 파란빛 테두리 영웅 | 보라빛 테두리·광채 전설 | 금색·주황빛 테두리·광채 신화 | 붉은빛·강한 주황 광채·이펙트 유일 | 무지개빛·특수 이펙트 각성 완료: 아이콘 주변 날개형 이펙트 또는 강한 빛 테두리 있음 각성 미완: 위 이펙트 없음 스킬 등급: 신화스킬 = 붉은빛/강한 이펙트 (빨강·주황) 전설스킬 = 금색·주황빛 영웅스킬 = 보라빛 ※ 전설스킬(전스)과 신화스킬(신스)은 별개. 반드시 구분할 것.
